# Session Handoff: 2026-05-14

## 1. Session Summary
**PaperQuantLab** is a local-first multi-asset research workstation. It is currently at a **functional prototype** level, emphasizing explainability and human-in-the-loop analysis. The system strictly follows a **simulation-first** philosophy, where no live brokerage execution is implemented or planned. The workstation serves as a sandbox for comparing market interpretations and strategy outcomes.

## 3. Current Architecture Snapshot
- **MarketProvider**: Decouples data sources (CCXT) from internal logic.
- **SimulatedMarket**: Manages virtual balances/positions; blocks real orders.
- **Signal Schema**: Now includes `persona_id`, `reasoning_trace`, `market_regime`, and `outcomes`.
- **Decision Journal**: JSON-based local persistence for all research data.
- **Outcome Tracking**: Background task periodically evaluates signal deltas.
- **Persona Framework**: Pluggable profiles (Conservative, Momentum, Contrarian, Risk Mgr).
- **UI Components**: Modular components for charts, order books, and persona experiments.

## 5. Important Constraints
- **Simulation-Only**: Brokerage execution logic is explicitly prohibited.
- **Local-First**: No cloud dependencies; all data stored in local JSON/SQLite.
- **Explainability-First**: Signals must have a `reason` or `reasoning_trace`.
- **Human-in-the-Loop**: Systems are research assistants, not autonomous traders.

## 6. Current Known Limitations
- **Tracking Gaps**: Outcome tracking only occurs while the backend is running.
- **Rate Limits**: Persona-heavy analysis is constrained by exchange API limits.
- **Error Handling**: Frontend requires more robust recovery from backend disconnection.
- **Backup**: No automated rotation or versioning for `decision_journal.json`.
