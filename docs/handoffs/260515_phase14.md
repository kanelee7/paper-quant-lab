# Session Handoff: Phase 14 - Research Workflow Presets & Guided Review Pipelines

## 1. Phase 14 Summary
This phase moved the workstation from "capability-rich" to **structured and guided**. We implemented a framework for Research Workflow Presets and Guided Review Pipelines, ensuring that researchers can reliably execute repeatable analytical studies and systematic evidence reviews.

## 2. Key Deliverables

### Research Workflow Preset Framework
- **`workflow_manager.py`**: A new core module that manages `ResearchWorkflowPreset` study templates.
  - **Initial Presets**:
    - **Volatility Regime Study**: Focuses on persona adaptation in volatile markets.
    - **Contrarian Failure Review**: Analyzes reversal logic blind spots in strong trends.
    - **Post-Mortem Outcome Review**: Standardized post-trade retrospective.
  - **Step-by-Step Guidance**: Every preset defines a series of logical research steps (e.g., "Filter Volatile" -> "Compare Reasoning" -> "Annotate").

### Guided Review UI
- **`ResearchWorkflowGuide.tsx`**: A new top-level sidebar component ("RESEARCH GUIDANCE").
  - **Pipeline Selector**: Allows researchers to choose a study template.
  - **Progress Tracking**: Visual indicators (Progress Bar + Step Badges) showing current status in the research pipeline.
  - **Contextual Help**: Provides specific guidance text for every step of the study.
  - **Mode Integration**: Automatically switches the workstation to the recommended mode (e.g., REVIEW mode) when starting a preset.

### Ergonomics & UI Refinements
- **`SignalJournal.tsx` Cleanup**: Refactored the journal to fix duplicated imports and improve modal readability.
- **Workflow-Aware Navigation**: Integrated the guide as the first thing a researcher sees in the sidebar to encourage structured work.

## 3. Verified Improvements
- **Repeatability**: Studies are now standardized, ensuring consistent data collection across different experiment runs.
- **Reduced Cognitive Overload**: Researchers no longer need to remember the "correct" review sequence; the platform guides them.
- **Improved Onboarding**: New users can immediately participate in complex analytical workflows through presets.

## 4. Operational Constraints
- **Heuristic Guidance**: Presets provide structural guidance; the actual analytical conclusions remain the responsibility of the human researcher.
- **State Persistence**: Current workflow progress is held in memory; refreshing the browser resets the active guide (future phases may persist this to JSON).
